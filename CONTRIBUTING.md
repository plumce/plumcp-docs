# Contributing to PluMCP

Thank you for considering to contribute to this project.

Please be respectful, inclusive, and constructive in all your interactions.

## How to contribute

You may contribute to this project in several ways:

#### Become a user

- Test and use PluMCP features in your applications
- Make innovative use of PluMCP in your applications
- Share your experience and success story
- Help others understand/adopt this project

#### Become an evangelist

- Create example code or write documentation
- Create writeup, illustrations or video
- Spread awareness about this project

#### Support the project

- Sponsor this project
- Test ambiguous use cases or corner areas of functionality
- Raise a bug or feature request with adequate details

#### Send a Pull Request

- Ensure a target issue is filed and discussed before raising PR
- Raise PRs from a fix or feature branch only
- Configure your editor to follow the whitespace and indentation used in
  the project, so that unwanted changes do not leak into the PR
- Follow Git commit rules: https://chris.beams.io/git-commit#seven-rules
- Ensure your PR does not break or undermine existing tests
- Keep PRs focused on the filed issue only, with adequate test coverage
- All PR contributions require a signed Contributor License Agreement (CLA)
  - Individual contributors sign automatically via GitHub
  - Contributions made as part of employment require a signed Corporate CLA
